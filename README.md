# Система продажу квитків на події 🎟️

Вебдодаток для управління заходами в актовому залі та бронювання квитків.
Проєкт реалізовано з використанням архітектури MVC.

## 🛠 Технології
* **Backend:** Node.js, Express.js
* **Frontend:** HTML, CSS, EJS (шаблонізатор)
* **Database:** PostgreSQL (пакет `pg`)

## 🚀 Як запустити проєкт локально

1. **Встановіть залежності:**
   Відкрийте термінал у папці з проєктом і виконайте команду:
   \`\`\`bash
   npm install
   \`\`\`

2. **Налаштуйте базу даних:**
   * Відкрийте pgAdmin та створіть порожню базу даних з назвою `hall_ticketing`.
   * Створіть файл `.env` у кореневій папці проєкту (скопіюйте структуру з `.env.example`) та впишіть свій пароль від бази даних.

3. **Запустіть сервер:**
   \`\`\`bash
   node server.js
   \`\`\`
   *(Таблиці в базі даних створяться автоматично при першому запуску).*

4. **Відкрийте сайт:**
   Перейдіть у браузері за адресою: [http://localhost:3000](http://localhost:3000)

## 💡 Функціонал
* **Клієнтський режим:** перегляд доступних заходів, перевірка вільних місць, бронювання квитків (із захистом від дублів).
* **Режим адміністратора:** створення нових подій, редагування та видалення існуючих (з обмеженням: заборонено редагувати/видаляти подію, якщо до неї залишилось менше 24 годин).
